    Exercices sur les Classes et Exceptions

    1. Le Compte Bancaire (Le Classique)

    Concept

    Gestion de l'argent avec interdiction de découvert et validation des montants.

    Champs privés

    • _solde (decimal)
    • _titulaire (string)

    Propriétés (Get/Set)

    • Titulaire:
      • Get: Retourne _titulaire.
      • Set: Si la valeur est vide, nulle ou contient moins de 2 caractères -> ArgumentException. Sinon, assigne _titulaire.
    • Solde:
      • Get: Retourne _solde.
      • Set: Privé (ou protégé). On ne modifie le solde que via les méthodes.

    Constructeur

    • Prend le titulaire et un montant initial. Utilise la propriété Titulaire pour valider le nom. Si le montant initial est < 0 -> ArgumentOutOfRangeException.

    Méthodes

    • Retirer(decimal montant):
      • Si montant <= 0 -> ArgumentException.
      • Si montant > _solde -> InvalidOperationException (Fonds insuffisants).
      • Sinon : _solde -= montant.

    2. Le Produit d'Inventaire (Validation de Format)

    Concept

    Un produit doit avoir un code spécifique et un prix cohérent.

    Champs privés

    • _codeProduit (string)
    • _prix (double)
    • _quantite (int)

    Propriétés (Get/Set)

    • CodeProduit:
      • Set: Doit commencer par "PROD-" et faire 10 caractères au total. Sinon -> FormatException.
    • Prix:
      • Set: Si valeur < 0.01 -> ArgumentOutOfRangeException (Un produit ne peut être gratuit ou négatif).
    • Quantite:
      • Set: Si valeur < 0 -> ArgumentException (Impossible d'avoir un stock négatif).

    Constructeur

    • Initialise les 3 propriétés.

    Méthodes

    • Vendre(int qteAchetee):
      • Si qteAchetee > _quantite -> InvalidOperationException (Stock insuffisant).
      • Sinon : _quantite -= qteAchetee.

    3. Le Personnage de RPG (Bornes Max/Min)

    Concept

    Gestion de points de vie (PV) qui ne peuvent ni dépasser le max, ni tomber sous 0.

    Champs privés

    • _pvActuels (int)
    • _pvMax (int)
    • _nom (string)

    Propriétés (Get/Set)

    • PvMax:
      • Set: Si valeur < 10 -> ArgumentOutOfRangeException (Minimum 10 PV pour exister). Assigne _pvMax.
    • PvActuels:
      • Set:
        • Si valeur < 0 -> Assigne 0 (Le perso est mort, on ne va pas à -50).
        • Si valeur > _pvMax -> Assigne _pvMax (On ne peut pas dépasser le max).
        • Sinon -> Assigne la valeur.

    Constructeur

    • Prend le nom et le PvMax. Initialise _pvActuels à la valeur de _pvMax.

    Méthodes

    • RecevoirDegats(int degats):
      • Si degats < 0 -> ArgumentException.
      • Algorithme : PvActuels -= degats (Le setter gérera le blocage à 0).
    • BoirePotion(int soin):
      • Algorithme : PvActuels += soin (Le setter gérera le blocage au Max).

    4. Le Thermostat Intelligent (Plages de confort)

    Concept

    Un appareil qui contrôle la température mais qui refuse les températures extrêmes pour la sécurité.

    Champs privés

    • _temperature (double)
    • _mode (string) – ex: "Chauffage", "Climatisation", "Arrêt".

    Propriétés (Get/Set)

    • Mode:
      • Set: Si la valeur n'est pas une des 3 chaînes autorisées -> ArgumentException ("Mode inconnu").
    • Temperature:
      • Set:
        • Si Mode est "Arrêt", on ne peut pas changer la température cible -> InvalidOperationException.
        • Si valeur < 10 ou valeur > 30 -> ArgumentOutOfRangeException (Sécurité thermique).
        • Sinon -> Assigne _temperature.

    Constructeur

    • Initialise le mode à "Arrêt" et la température à 20.

    Méthodes

    • ForcerModeEco():
      • Algorithme : Passe le mode à "Chauffage" et met la température à 18.

    5. L'Ascenseur (Logique spatiale)

    Concept

    Un ascenseur qui ne peut pas aller plus bas que le sous-sol ou plus haut que le toit.

    Champs privés

    • _etageActuel (int)
    • _etageMin (int) - ex: -2
    • _etageMax (int) - ex: 10

    Propriétés (Get/Set)

    • EtageActuel:
      • Get: Retourne l'étage.
      • Set: Privé. Seule la méthode de déplacement peut changer l'étage.

    Constructeur

    • Définit min et max. Si min > max -> ArgumentException (Configuration impossible).
    • Initialise EtageActuel à 0 (Rez-de-chaussée).

    Méthodes

    • AllerA(int etageCible):
      • Validation 1 : Si etageCible < _etageMin ou etageCible > _etageMax -> ArgumentOutOfRangeException (Cet étage n'existe pas).
      • Validation 2 : Si etageCible == _etageActuel -> InvalidOperationException (Vous y êtes déjà).
      • Algorithme (Simulation) :
        • Tant que _etageActuel < etageCible, incrémenter _etageActuel.
        • Tant que _etageActuel > etageCible, décrémenter _etageActuel.
        • (Optionnel : Afficher "Ding!" à chaque étage).

    Résumé des Exceptions utilisées :

    • ArgumentException: L'argument est invalide (ex: format de chaîne, stock négatif).
    • ArgumentOutOfRangeException: L'argument est un chiffre hors des limites acceptables (ex: note de 150%).
    • FormatException: Le format du texte ne respecte pas le masque (ex: Code produit).
    • InvalidOperationException: L'état actuel de l'objet interdit l'action (ex: retirer de l'argent sans solde, changer la température quand c'est éteint).
